# 日本語ドキュメント作成ガイドライン

## 基本方針

- すべての spec ファイル、コメント、ドキュメントは日本語で記述する
- コード内のコメントも日本語で記述する
- 変数名や関数名は英語でも構わないが、説明やドキュメントは日本語とする

## 技術用語の扱い

- AWS サービス名は英語のまま使用（例：Lambda、S3、DynamoDB）
- 一般的な技術用語は日本語化（例：要件 → Requirements、設計 → Design）
- CDK 関連の専門用語は英語のまま使用し、必要に応じて日本語で説明を併記

## ファイル構成

- requirements.md：要件定義書
- design.md：設計書
- tasks.md：実装タスク一覧

すべて日本語で記述し、読みやすさを重視する。
